Electron 到 Tauri 迁移方案

桌面 Todo 组件的技术栈转型

现状：Electron (Node.js)

核心架构：

  • 主进程 (JS): 处理 IPC, 窗口生命周期, 文件读写
  • 渲染进程 (React): UI 渲染, 调用 electronAPI
  • 存储: fs.writeFileSync (JSON)

置顶控制:

setAlwaysOnTop(true, 'screen-saver')

避开 Win+D，常驻桌面

目标：Tauri (Rust)

核心架构：

  • 后端 (Rust): 处理 IPC (Commands), 系统托盘, 线程
  • 前端 (React): UI 渲染, 调用 @tauri-apps/api
  • 存储: serde_json + Rust IO (更安全)

置顶控制:

window.set_always_on_top(true)

Win+D 避让需要底层 Rust 实现 (Win32 API)

推荐迁移路径 (方案 2)

优势

  • 体积缩小 90%: 从 ~200MB 减至 ~10MB
  • 内存占用降低: 消除冗余的 Chrome 实例
  • 安全性更高: 前端无法直接写文件，需经后端授权

挑战

  • Rust 语法: 需要编写少量的 Rust 胶水代码
  • Windows 特定行为: "置顶避开 Win+D" 需要 Win32 调用
1

完全保留 (Win32 API)

在 Rust 中调用 Windows 原生接口，确保小组件行为与 Electron 完全一致。

2

简化方案 (Tauri 标准 API)

普通的始终置顶。按 Win+D 会随桌面一同隐藏，手动点击托盘或任务栏恢复。
